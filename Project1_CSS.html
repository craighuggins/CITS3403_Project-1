Cascading Style Sheets

The purpose and motivations of CSS

Introduction

Prior to the availability of Cascading Style Sheets (CSS), all site appearance properties were handled by the HTML code. As sites began to embody a wide variety of styles, the HTML was becoming increasingly complex. To address this, CSS was introduced as a way to separate style from content. It provided more control to web developers by allowing multiple web pages to share the same formatting through one accompanying CSS document, stored in a separate .css file. This saves a lot of time, and provides advantages in site maintenance. CSS is prominantly used to give websites a uniform appearance theme throughout.

The CSS document is a set of declarations that define how elements on the page are to be displayed. The syntax of a declaration block is as follows: selector { property : value; } Where 'selector' is the element to apply the style to, 'property' is the property we want to change, and 'value' is what we want to change the property to. See below for an example.

Example

Inside the declaration block, you could have as many declarations as you want. For example, this web site has the same font and a constant light background colour on all pages. The rules have been applied to the <body> element, so it covers everything visible to the user:
body { background-color: #F7F9F9; font-family: 'Exo 2', sans-serif; }

Selectors

The selector can be used for Elements, Classes, ID's, or a combination. This gives great choice as to which elements on a page inherit a particular style. Usually, we want to give certain elements a particular appearance and format, and for this we have class selectors and ID selectors.

Class

Class selectors are useful for grouping multiple elements together that we want to share format and appearance. Rather than applying the style individually, we group elements by their class, and apply the class inside the element's tag as an attribute. It is declared in CSS by placing a dot before the selector: .selector { property : value; }

Example

All of these example boxes are using the same class: example_long
Which specifies the size and shape of the box, the font inside, margins, colour and box shadow. The purpose was to give these boxes separation from the main text content of the page and draw the attention of the user. It has been applied to the element with the syntax: <div class="example_long" id="Example_2">

ID

ID selectors target elements with a specific ID attribute value. They are useful to giving a single element of the page a unique identity, setting it apart from the rest. It is declared using a hash tag before the element. #selector { property : value; }

Example

This example box has the same class; example_long; however it has been assigned the ID dark. The dark CSS ID selector has been defined as: #dark { background-color: #474444; color: #eee; }

Pseudo-classes

Pseudo selectors define an element when it is in a special state, such as when the mouse hovers over it, or the element is in focus. It is useful in links, which can have a different appearance when it has been visited, is currently active, or unvisited. The syntax to use is: selector:pseudo-class { property : value; }

Example

You will have noticed that links throughout this website change when the mouse hovers over them. The links in-text underline, the table of contents links turn a light blue colour, and the navigation bar links have an aqua glow with white text. These effects have been set through pseudo-classes.

Selector Combinations

'Combinators' can be used to achieve precise selector behaviour. We can choose to select elements that are inside other elements, such as paragraph elements that are inside a div: div p { background-color: yellow; }

Example

Looking at the dark-coloured footer on each page, the text colour is white as opposed to black everywhere else. This has been set by defining the <p> element inside the div with class 'footer' to have white font: .footer p { color: white; }

Formats

CSS has 3 different formats:

  • Inline
  • Internal
  • External

Inline

The inline method applies a style directly to an HTML element and is typically avoided for breaking the rule of separating style from content

Internal

Internal CSS is placed inside the <style> tags in the <head> section of the HTML document. It will only apply to a single document, so it can be useful to separate one particular page's style from all others.

External

External CSS to carry style information over various pages. All style rules are stored in a separate .css file that is linked to in the <head> section of an HTML document.

Example

You can see that across this website, all pages have a similar colour scheme, font, and layout. This has been set up through one single external CSS document called 'Project1_CSS.css' which contains all style information, and has been linked on every page.
